MissionWeaveProtocol ist ein Group-orientiertes Kooperationsprotokoll für
autonome Agent innerhalb einer vertrauenswürdigen Organization. Ein Agent kann
an vielen Mission Group teilnehmen, dauerhafte Message mit Peers austauschen,
ausdrückliche WorkItem in Group-spezifische Warteschlangen aufnehmen und die
Arbeit über mehrere Group nach seiner eigenen lokalen Richtlinie planen.

## Kurzfassung

1. Ein menschlicher MissionOwner erstellt eine begrenzte Mission mit
   Abschlussdefinition, Frist, Budget und Approval-Richtlinie.
2. Die Mission erhält genau eine Group, eine dauerhafte Event-Historie und einen
   austauschbaren Coordinator.
3. Mitglieder der Group besprechen den Kontext und können Arbeit vorschlagen,
   doch Message autorisieren niemals eigenständig eine Ausführung.
4. Ein Coordinator oder ein ausdrücklich begrenzter Delegierter wandelt einen
   Vorschlag in ein WorkItem mit einem maschinenlesbaren Work Contract um.
5. Ein Worker führt Admission Control durch, nimmt ein Angebot in seine
   Group-spezifische Warteschlange auf und entscheidet mit seinem globalen
   Scheduler, wann es ausgeführt wird.
6. Die Arbeit erzeugt inhaltsadressierte Artifact und den Akzeptanzkriterien
   zugeordnete Evidence.
7. Der Coordinator verifiziert die Ergebnisse des Worker, bevor der menschliche
   MissionOwner eine genaue Mission-Revision und einen genauen Satz von Artifact
   genehmigt.

## Protokollgrenzen

MissionWeaveProtocol definiert weder Agent-to-Agent RPC noch
organisationsübergreifende Föderation, physisches Peer-to-Peer-Routing,
verteilten Konsens, Modell-Prompts oder private Modellüberlegungen. Eine
Organization stellt Identität, Richtlinien, Autorisierung, dauerhafte
Group-Reihenfolge und menschliche Verantwortlichkeit bereit.

:::note[Conversation liefert Kontext, keine Autorität]

Ein Message, eine Erwähnung, eine Zusammenfassung, ein Context Package oder eine
Modellausgabe kann allein weder einen Tool-Aufruf, eine geschäftliche Aktion,
die Zuweisung eines WorkItem, eine Budgetausgabe noch die Vergabe einer
Berechtigung autorisieren. Folgenschwere Aktionen erfordern ausdrücklichen
strukturierten Zustand.

:::
