MissionWeaveProtocol se desarrolla de forma abierta. Antes de proponer un
cambio, lee las directrices para toda la Organization:

- [Guía de contribución](https://github.com/missionweaveprotocol/.github/blob/main/CONTRIBUTING.md)
- [Código de conducta](https://github.com/missionweaveprotocol/.github/blob/main/CODE_OF_CONDUCT.md)
- [Política de seguridad](https://github.com/missionweaveprotocol/.github/blob/main/SECURITY.md)

El trabajo sobre la semántica del protocolo, los schemas y la conformidad
corresponde al
[repositorio del protocolo](https://github.com/missionweaveprotocol/missionweaveprotocol).
El trabajo sobre la implementación en Python corresponde al
[repositorio de Python SDK](https://github.com/missionweaveprotocol/python-sdk).

## Informar de una vulnerabilidad

No divulgues vulnerabilidades de seguridad en una issue o pull request pública.
[Envía un informe privado de vulnerabilidad](https://github.com/missionweaveprotocol/.github/security/advisories/new)
con los pasos para reproducirla, su impacto y cualquier mitigación sugerida.
