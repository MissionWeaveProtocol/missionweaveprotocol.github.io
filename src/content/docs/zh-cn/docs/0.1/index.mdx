MissionWeaveProtocol 是一种面向 Group 的协作协议，适用于在同一个受信任 Organization 内工作的自主 Agent。Agent 可以参与多个 Mission
Group，与对等方交换持久 Message，将明确的 WorkItem 接收到各 Group 的队列中，并依据自身的本地策略跨 Group 调度工作。

## 简要说明

1. 人类 MissionOwner 创建一个边界明确的 Mission，并设定完成定义、截止时间、预算和批准策略。
2. 该 Mission 恰好获得一个 Group、一份持久 Event 历史和一个可替换的 Coordinator。
3. Group 成员可以讨论上下文并提出工作，但 Message 本身绝不会授权执行。
4. Coordinator 或获得明确范围授权的受委托者将提议转化为带有机器可读 Work
   Contract 的 WorkItem。
5. Worker 执行准入控制，将邀约接收到自身的 Group 队列中，并由其全局 Scheduler 决定何时执行。
6. 工作产出按内容寻址的 Artifact，以及映射到验收标准的 Evidence。
7. Coordinator 先验证 Worker 结果，再由人类 MissionOwner 批准准确的 Mission 修订版本和 Artifact 集合。

## 协议边界

MissionWeaveProtocol 不定义 Agent-to-Agent
RPC、跨 Organization 联邦、物理点对点路由、分布式共识、模型提示词或模型私有推理。一个 Organization 负责提供身份、策略、授权、持久 Group 排序和人类问责机制。

:::note[对话提供上下文，不提供权限]

Message、提及、摘要、Context
Package 或模型输出本身不能授权工具调用、业务操作、WorkItem 分派、预算支出或权限授予。会产生后果的操作必须依赖明确的结构化状态。

:::
