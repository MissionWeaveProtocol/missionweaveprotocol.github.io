MissionWeaveProtocol は、1 つの信頼された Organization 内で働く自律 Agent のための Group 指向協働プロトコルです。Agent は複数の Mission
Group に参加し、他の Agent と永続的な Message を交換し、明示的な WorkItem を Group ごとのキューに受け入れ、自身のローカルポリシーに従って Group を横断する作業をスケジュールできます。

## 要点

1. 人間の MissionOwner が、完了条件、期限、予算、承認ポリシーを備えた、範囲の定められた Mission を作成します。
2. Mission には、正確に 1 つの Group、1 つの永続的な Event 履歴、交換可能な Coordinator が割り当てられます。
3. Group メンバーはコンテキストを話し合い、作業を提案できますが、Message だけで実行が認可されることはありません。
4. Coordinator または明示的に範囲を与えられた委任先が、提案を機械可読の Work
   Contract を持つ WorkItem に変換します。
5. Worker は受入判定を行い、Work
   Offer を Group ごとのキューに受け入れ、全体を管理する Scheduler を通じていつ実行するかを決定します。
6. 作業の結果としてコンテンツアドレス指定された Artifact と、受け入れ基準に対応付けられた Evidence が生成されます。
7. Coordinator が Worker の結果を検証した後、人間の MissionOwner が正確な Mission リビジョンと Artifact セットを承認します。

## プロトコルの境界

MissionWeaveProtocol は、Agent 間 RPC、Organization をまたぐフェデレーション、物理的なピアツーピアルーティング、分散合意、モデルプロンプト、モデルの非公開推論を定義しません。1 つの Organization が、アイデンティティ、ポリシー、認可、永続的な Group の順序付け、人間による説明責任を提供します。

:::note[Conversation はコンテキストであり、権限ではありません]

Message、メンション、要約、Context
Package、モデル出力だけで、ツール呼び出し、業務操作、WorkItem の割り当て、予算支出、権限付与を認可することはできません。結果を伴う操作には、明示的な構造化状態が必要です。

:::
