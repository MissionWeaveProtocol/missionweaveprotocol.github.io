MissionWeaveProtocol est développé de manière ouverte. Avant de proposer une
modification, consultez les règles qui s’appliquent à toute l’Organization :

- [Guide de contribution](https://github.com/missionweaveprotocol/.github/blob/main/CONTRIBUTING.md)
- [Code de conduite](https://github.com/missionweaveprotocol/.github/blob/main/CODE_OF_CONDUCT.md)
- [Politique de sécurité](https://github.com/missionweaveprotocol/.github/blob/main/SECURITY.md)

Les travaux sur la sémantique du protocole, les schémas et la conformité doivent
être réalisés dans le
[dépôt du protocole](https://github.com/missionweaveprotocol/missionweaveprotocol).
Les travaux d’implémentation relèvent du dépôt du SDK officiel correspondant ;
consultez la [vue d’ensemble des SDK](../sdk/) pour choisir Python, Go,
TypeScript, Java, Rust ou C++.

## Signaler une vulnérabilité

Ne divulguez pas une vulnérabilité de sécurité dans une issue ou une pull
request publique.
[Envoyez un rapport de vulnérabilité privé](https://github.com/missionweaveprotocol/.github/security/advisories/new)
avec les étapes de reproduction, l’impact et toute mesure d’atténuation
proposée.
